====================================================
        NAPS2 WebScan LocalService v2.0.0
         Scanner TWAIN to HTTP/eSCL Bridge
====================================================

BEM-VINDO!
==========

Este pacote transforma seus scanners TWAIN em scanners
acessiveis via HTTP, permitindo scan direto do navegador.


INSTALACAO RAPIDA (3 PASSOS)
=============================

1. EXECUTAR INSTALADOR
   - Clique com botao direito em "instalador.bat"
   - Selecione "Executar como administrador"

2. INSTALAR SERVICO
   - Digite 1 e pressione ENTER

3. INICIAR SERVICO  
   - Digite 3 e pressione ENTER

PRONTO! Seus scanners agora estao acessiveis via HTTP.


VERIFICAR INSTALACAO
=====================

Abra seu navegador em:
  http://localhost:5000/api/scanners

Deve retornar uma lista JSON com seus scanners.

  5. Ver scanners disponiveis
  6. Abrir API no navegador
  0. Sair


API REST
========

Quando o servico estiver rodando, use:

Listar scanners:
  GET http://localhost:5000/api/scanners

Scanner atual:
  GET http://localhost:5000/api/scanners/current

Selecionar scanner:
  POST http://localhost:5000/api/scanners/{id}/select


PROTOCOLO eSCL
==============

Cada scanner recebe sua propria porta eSCL:

Scanner 1: http://localhost:9880/eSCL/
Scanner 2: http://localhost:9881/eSCL/
Scanner 3: http://localhost:9882/eSCL/

Use essas URLs para fazer scan via protocolo eSCL
(compativel com AirPrint).


REQUISITOS
==========

- Windows 10/11 ou Windows Server 2016+
- .NET 8.0 Runtime (https://dotnet.microsoft.com/download/dotnet/8.0)
- Drivers TWAIN dos scanners instalados
- Permissoes de Administrador (para instalar servico)


SOLUCAO DE PROBLEMAS
=====================

Scanners nao aparecem:
  1. Verifique se drivers TWAIN estao instalados
  2. Teste scanner no software nativo
  3. Reinicie o servico (opcao 2 depois opcao 1)

Servico nao inicia:
  1. Verifique se porta 5000 esta livre
  2. Veja logs: Event Viewer > Application
  3. Execute: netstat -ano | findstr 5000

Erro ao instalar:
  1. Execute instalador.bat como Administrador
  2. Verifique se .NET 8 Runtime esta instalado
  3. Tente instalacao manual (veja INSTALACAO_SERVICO.md)


DESINSTALAR
===========

1. Executar instalador.bat como Administrador
2. Opcao 2 - Desinstalar servico
3. Deletar esta pasta
